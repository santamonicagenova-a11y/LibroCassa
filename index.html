Budget Ristorante · Vercel + Neon

Configurazione

La pagina gira su Vercel e chiama il database Neon tramite una funzione serverless — nessuna credenziale nel browser.

Architettura

Browser → POST /api/sql → Vercel Function → HTTPS → Neon Postgres
✓ Nessun problema CORS — browser e API sono sullo stesso dominio Vercel
✓ Credenziali Neon nelle variabili d'ambiente di Vercel, mai nel browser
✓ Tutto nella stessa repo GitHub già creata

Guida configurazione — 10 minuti

Aggiungi api/sql.js alla tua repo GitHub
Nella repo ristorante-cassa su GitHub:
→ Clicca "Add file" → "Create new file"
→ Nome file: api/sql.js (scrivi esattamente così, con lo slash — GitHub creerà la cartella api automaticamente)
→ Incolla il contenuto del file worker.js che hai scaricato (è già il codice giusto per Vercel, nonostante il nome)

Schema SQL — incolla nel SQL Editor di Neon

-- Un record per ogni giornata operativa del ristorante
CREATE TABLE IF NOT EXISTS incassi (
  id          UUID          PRIMARY KEY DEFAULT gen_random_uuid(),
  data        DATE          NOT NULL UNIQUE,
  pranzo      NUMERIC(10,2),   -- incasso pranzo
  cena        NUMERIC(10,2),   -- incasso cena
  totale      NUMERIC(10,2),   -- totale giornata
  corrisp     NUMERIC(10,2),   -- corrispettivi
  pos         NUMERIC(10,2),   -- POS / carta
  banca       NUMERIC(10,2),   -- bonifici
  cassa       NUMERIC(10,2),   -- contante
  ft          NUMERIC(10,2),   -- fatture
  cp          SMALLINT,        -- coperti pranzo
  cs          SMALLINT,        -- coperti sera
  media_p     NUMERIC(8,2),    -- scontrino medio pranzo
  media_s     NUMERIC(8,2),    -- scontrino medio sera
  note        TEXT,
  created_at  TIMESTAMPTZ DEFAULT now()
);
CREATE INDEX IF NOT EXISTS idx_ig_data ON incassi(data);
CREATE INDEX IF NOT EXISTS idx_ig_anno ON incassi(EXTRACT(YEAR FROM data));
CREATE INDEX IF NOT EXISTS idx_ig_dow  ON incassi(EXTRACT(DOW FROM data));
